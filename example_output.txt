BOARD OCCUPANCY STATUS
----------------------

### Test Case 1: Verify Occupied Square Returns True
**Test Case Title:** Check Occupied Square Status  
**Steps:**  
1. Place a game piece on a specific square on the board (e.g., A1).  
2. Call the method to check the occupancy status of the square A1.  
**Expected Outcome:** The system should return true, indicating that the square A1 is occupied.

---

### Test Case 2: Verify Empty Square Returns False
**Test Case Title:** Check Empty Square Status  
**Steps:**  
1. Ensure that a specific square on the board (e.g., B2) is empty (no pieces placed).  
2. Call the method to check the occupancy status of the square B2.  
**Expected Outcome:** The system should return false, indicating that the square B2 is empty.

---

### Test Case 3: Verify Multiple Occupied Squares
**Test Case Title:** Check Multiple Occupied Squares  
**Steps:**  
1. Place game pieces on multiple squares (e.g., C3, D4, and E5).  
2. Call the method to check the occupancy status of each of the squares C3, D4, and E5.  
**Expected Outcome:** The system should return true for C3, D4, and E5, indicating that all these squares are occupied.

---

### Test Case 4: Verify Mixed Occupancy Status
**Test Case Title:** Check Mixed Occupancy Status  
**Steps:**  
1. Place a game piece on square F6.  
2. Ensure that square G7 is empty.  
3. Call the method to check the occupancy status of both squares F6 and G7.  
**Expected Outcome:** The system should return true for F6 and false for G7, indicating that F6 is occupied while G7 is empty.

---

### Test Case 5: Verify Occupancy Status After Removing a Piece
**Test Case Title:** Check Status After Removing a Piece  
**Steps:**  
1. Place a game piece on square H8.  
2. Call the method to check the occupancy status of square H8.  
3. Remove the game piece from square H8.  
4. Call the method again to check the occupancy status of square H8.  
**Expected Outcome:** The system should return true after step 2 and false after step 4, indicating that the square was occupied and then became empty.

---

### Test Case 6: Verify Occupancy Status on Unused Square
**Test Case Title:** Check Status of Unused Square  
**Steps:**  
1. Identify a square on the board that has never had a piece placed on it (e.g., A8).  
2. Call the method to check the occupancy status of square A8.  
**Expected Outcome:** The system should return false, indicating that square A8 is empty.

---

### Test Case 7: Verify Occupancy Status with Invalid Input
**Test Case Title:** Check Status with Invalid Square Reference  
**Steps:**  
1. Attempt to check the occupancy status of a square that does not exist (e.g., Z9).  
**Expected Outcome:** The system should handle the invalid input gracefully, either by returning an error message or a specific response indicating that the square is invalid.

---

### Test Case 8: Verify Occupancy Status After Multiple Changes
**Test Case Title:** Check Status After Multiple Changes  
**Steps:**  
1. Place a game piece on square B1.  
2. Call the method to check the occupancy status of square B1.  
3. Remove the piece from square B1.  
4. Place a game piece on square C2.  
5. Call the method to check the occupancy status of both squares B1 and C2.  
**Expected Outcome:** The system should return true for C2 and false for B1, indicating that B1 is empty and C2 is occupied.

GAME BOARD MANAGEMENT
---------------------

### Test Case 1: Assign a Piece to an Empty Square
**Test Case Title:** Assign a Piece to an Empty Square  
**Steps:**  
1. Start a new game session.  
2. Identify an empty square on the game board.  
3. Select a game piece to occupy the identified square.  
4. Execute the action to place the selected piece on the empty square.  
**Expected Outcome:**  
The selected piece should occupy the identified square, and the square should reflect the presence of the piece.

---

### Test Case 2: Attempt to Assign a Piece to an Occupied Square
**Test Case Title:** Attempt to Assign a Piece to an Occupied Square  
**Steps:**  
1. Start a new game session.  
2. Identify a square that is already occupied by another piece.  
3. Select a different game piece to occupy the identified square.  
4. Execute the action to place the selected piece on the occupied square.  
**Expected Outcome:**  
The action should be rejected, and the original piece should remain in the square. An appropriate error message should be displayed indicating that the square is already occupied.

---
